# 轻量实验：裸 LLM vs 带模板 LLM

## 模板提示词
```text
## 第3章 处置程序
生成要求：按先后顺序组织程序，必要时拆分阶段

### 3.1 信息报告
生成要求：根据事件场景生成报告对象、报告内容和报告时限

#### 3.1.1 报告对象
生成要求：列出应报告的调度、运维、安监、管理和外协对象

#### 3.1.2 报告内容
生成要求：说明需要报告的事件信息、影响范围、风险和现场需求

#### 3.1.3 报告时限
生成要求：明确首报、续报和终报时限

### 3.2 先期处置
生成要求：优先组织隔离、停运、警戒、排险等先期动作

#### 3.2.1 现场隔离
生成要求：明确隔离区域、隔离方式和防止扩大事故的要求

#### 3.2.2 停运或限制运行
生成要求：根据设备状态生成停运、转供、限负荷或限制运行要求

#### 3.2.3 警戒与排险
生成要求：根据现场风险生成警戒、疏散和排险动作

### 3.3 现场研判
生成要求：结合故障原因、故障现象和现场条件生成研判要点

#### 3.3.1 原因研判
生成要求：围绕图谱故障原因和现场条件推断主要诱因

#### 3.3.2 现象核验
生成要求：核验当前现象与典型故障现象是否一致

#### 3.3.3 风险判定
生成要求：结合风险和现场状态判定作业风险与设备风险

### 3.4 抢修恢复
生成要求：结合应对措施生成抢修和恢复步骤

#### 3.4.1 抢修准备
生成要求：整理抢修前需要的人员、物资、工器具和作业条件

#### 3.4.2 现场抢修
生成要求：按顺序生成现场抢修动作

#### 3.4.3 恢复运行
生成要求：明确恢复供电、恢复运行和恢复后观察要求

### 3.5 响应终止
生成要求：根据风险解除、设备恢复和现场验收条件生成终止要求
```
