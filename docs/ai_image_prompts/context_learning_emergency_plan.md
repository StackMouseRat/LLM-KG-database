# 上下文学习在应急预案系统中的应用

## 完整绘图提示词

请生成一张严谨、清晰、适合学术论文使用的中文信息图，标题为“上下文学习在应急预案系统中的应用”。画面采用极简扁平化设计，白色背景，科技蓝与浅灰色配色，黑色黑体中文字体，排版整齐，线条清晰，整体风格类似教材或学术论文插图。

画布比例建议为 16:9 或 A4 横版。图中从上到下分为三个等高的横向矩形区域，每个区域用浅灰色细边框分隔，三个区域分别表示 Zero-shot、One-shot、Few-shot。每个区域采用“左侧方法名称 + 中间提示词结构 + 右侧输出结果”的布局。箭头统一使用科技蓝色，从中间内容指向右侧输出。

顶部区域标题为：
“Zero-shot（零样本）”

顶部区域左侧放置方法标签：
“Zero-shot（零样本）”
“无示例，直接根据任务与上下文生成”

顶部区域中间放置一个浅蓝色圆角矩形框，框内文字为：
“任务：生成应急预案章节”
“用户问题：站内光缆经过断路器间隔后通信中断，保护通道退出”
“图谱素材：故障原因、现象、处置措施、安全风险”
“章节模板：第4章 处置措施”
“提示：根据以上上下文生成当前章节内容”

顶部区域右侧放置输出框，文字为：
“输出：预案章节正文”

中间区域标题为：
“One-shot（单样本）”

中间区域左侧放置方法标签：
“One-shot（单样本）”
“提供一个示例，引导输出格式”

中间区域中间放置两个上下堆叠的圆角矩形框。上方为示例框，浅灰底，文字为：
“示例：”
“输入：今天吃什么”
“输出：irrelevant”
“提示：请输入电力设备故障、告警、检修或应急处置相关问题”

下方为当前任务框，浅蓝底，文字为：
“当前输入：站内光缆经过断路器间隔后通信中断”
“请判断是否进入后续预案生成链路”

中间区域右侧放置输出框，文字为：
“输出：边界判定标签”
“result = ok”

底部区域标题为：
“Few-shot（少样本）”

底部区域左侧放置方法标签：
“Few-shot（少样本）”
“提供多个示例，学习类别边界”

底部区域中间放置一个较大的浅蓝色圆角矩形框，框内按列表排列多个示例，文字为：
“示例1：今天吃什么 → irrelevant”
“示例2：发电机转子接地故障 → unsupported_device”
“示例3：变压器拒动 → incompatible_device_fault”
“示例4：主变旁断路器拒动 → ok”
“示例5：线路侧避雷器损坏 → ok”
“当前输入：站内光缆经过断路器间隔后通信中断，保护通道退出 →”

底部区域右侧放置输出框，文字为：
“输出：result = ok”
“message = 空”

图中可以使用简单的小图标辅助表达，但不要喧宾夺主：
Zero-shot 可使用“文档生成”图标；
One-shot 可使用“单个示例卡片”图标；
Few-shot 可使用“多张示例卡片”图标。
图标需保持线性、简洁、科技蓝风格。

要求：
1. 所有中文文字必须清晰可读，不要出现乱码、错字或变形文字。
2. 不要使用复杂背景、渐变过多、三维效果、卡通风格或装饰性元素。
3. 不要加入真实人物、设备照片、复杂电网照片。
4. 不要把图画成流程图软件截图，应保持学术插图风格。
5. 三个区域大小一致，间距均匀，层次清楚。
6. 重点突出三种上下文学习方式的差异：无示例、一个示例、多个示例。
7. 图中英文术语保持为 Zero-shot、One-shot、Few-shot，中文解释使用“零样本、单样本、少样本”。
8. 输出画面应适合直接放入论文，底部可添加小号图注：“图 2-3 上下文学习在应急预案系统中的应用”。
