# 4.1 数据与知识层设计信息

本文档整理第4章“数据与知识层设计”所需事实材料，覆盖 Nebula 图数据库、模板图空间、案例库、评估题库和历史结果文件。内容用于支撑论文中对系统底层知识组织方式的描述。

## 信息来源

- `docker/frontend-proxy/services/template_service.py`
- `docker/frontend-proxy/services/case_search_service.py`
- `docker/frontend-proxy/services/evaluation_question_service.py`
- `docker/frontend-proxy/services/experiment_service.py`
- `scripts/run_parallel_generation_pipeline.py`
- Nebula HTTP 网关：`http://127.0.0.1:8787/graph/query`
- 既有数据库核对结果和第4章前序整理记录

## 数据层组成

| 数据对象 | 存储位置 | 主要用途 | 调用方 |
|---|---|---|---|
| 设备故障知识图谱 | Nebula 多个设备图空间 | 存储故障类型、原因、现象、后果、风险、措施和资源。 | FastGPT 基本信息插件、图谱溯源服务、多故障查询插件。 |
| 预案模板 | Nebula `llmkg_templates` | 存储模板版本、章节层级、内容来源、固定文本和生成要求。 | 模板切片插件、模板查看与编辑接口。 |
| 案例知识库 | FastGPT 知识库数据集 | 检索与当前故障问题相似的历史案例。 | 代理层 `case_search_service.py`。 |
| 评估题库 | Nebula `llmkg_evaluation` | 存储实验套件、题组和题目。 | 实验页面与结构化评估服务。 |
| 流水线运行结果 | 文件目录 | 保存每次生成的 JSON 与 Markdown 结果。 | Python 流水线、前端历史恢复和实验分析。 |
| 模板默认快照 | 文件 `template_defaults_snapshot.json` | 支持模板章节配置回滚。 | 模板服务。 |

## 设备故障图谱结构

以 `llmkg_breaker` 为例，设备故障图谱采用统一的实体类型和业务关系边。其他设备图空间遵循相近建模方式。

### 实体类型

| Tag | 含义 | 生成链路用途 |
|---|---|---|
| `root_node` | 设备或图谱根节点 | 表达设备图谱入口。 |
| `fault_l1` | 一级故障分类 | 组织故障目录。 |
| `fault_l2` | 二级故障类型 | 基本信息插件匹配的核心故障节点。 |
| `fault_cause` | 故障原因 | 作为 `[KG]` 原因素材。 |
| `fault_symptom` | 故障现象 | 作为 `[KG]` 事件特征素材。 |
| `fault_consequence` | 故障后果 | 作为 `[KG]` 影响范围和危害素材。 |
| `safety_risk` | 安全风险 | 作为 `[KG]` 安全注意事项素材。 |
| `response_measure` | 处置措施 | 作为 `[KG]` 应急处置措施素材。 |
| `emergency_resource` | 应急资源 | 作为 `[KG]` 应急保障和物资清单素材。 |
| `entity` | 通用实体 | 兼容导入或历史通用实体。 |

### 节点字段

业务实体节点字段保持统一，便于通用查询、图谱展示和清洗逻辑复用。

| 字段 | 类型 | 含义 |
|---|---|---|
| `name` | string | 节点名称。 |
| `node_desc` | string | 节点描述。 |
| `lvl` | int64 | 节点层级或业务层级。 |
| `source_id` | int64 | 数据来源编号。 |
| `degree` | int64 | 节点度数或重要性辅助字段。 |
| `weight` | int64 | 权重字段。 |
| `stroke` | string | 前端展示或样式辅助字段。 |

### 关系边

| Edge | 含义 |
|---|---|
| `has_fault_category` | 设备根节点包含一级故障分类。 |
| `contains` | 一级故障包含二级故障，或通用包含关系。 |
| `caused_by` | 故障由某原因导致。 |
| `has_symptom` | 故障具有某现象。 |
| `results_in` | 故障导致某后果。 |
| `has_risk` | 故障关联某安全风险。 |
| `handled_by` | 故障由某处置措施处理。 |
| `needs_resource` | 故障处置需要某应急资源。 |
| `rel` | 历史通用关系边，包含 `relation` 字段。 |

多数业务边不设置额外属性，而是通过边类型直接表达语义。这样可以降低图查询和生成素材清洗的复杂度，也便于将查询结果按原因、现象、措施、后果、风险和资源分类输出。

## 模板图空间

预案模板存储在 `llmkg_templates`，默认模板 ID 为 `tpl_default_emergency`。模板服务默认访问：

| 配置项 | 默认值 |
|---|---|
| `TEMPLATE_GRAPH_URL` | `http://host.docker.internal:8787/graph/query` |
| `TEMPLATE_SPACE` | `llmkg_templates` |
| `TEMPLATE_ID` | `tpl_default_emergency` |

### 模板实体与关系

| Tag 或 Edge | 类型 | 含义 |
|---|---|---|
| `template` | Tag | 模板主记录。 |
| `template_version` | Tag | 模板版本。 |
| `template_section` | Tag | 模板章节节点。 |
| `template_binding` | Tag | 模板与设备或场景的绑定关系配置。 |
| `template_prompt_config` | Tag | 质量优化和评估提示词配置。 |
| `has_version` | Edge | 模板拥有版本。 |
| `has_section` | Edge | 模板版本拥有章节。 |
| `has_child_section` | Edge | 章节之间的父子层级。 |
| `binds_to` | Edge | 模板绑定设备或场景。 |

### 模板章节字段

| 字段 | 含义 |
|---|---|
| `section_no` | 章节编号。 |
| `title` | 章节标题。 |
| `level` | 章节层级。 |
| `order_no` | 排序编号。 |
| `source_type` | 内容来源类型，如固定文本、图谱生成或模型生成。 |
| `kg_field` | 对应图谱素材字段，如 `response_measure`。 |
| `fixed_text` | 模板固定文本。 |
| `gen_instruction` | 当前章节生成要求。 |

默认预案模板包含六个一级章节：事件特征、应急组织与职责、处置程序、处置措施、应急保障和附件。模板切片插件会读取这些章节及其子章节，并转换为面向生成模型的章节模板文本。

## 评估题库与实验数据

评估题库存储在 `llmkg_evaluation`。当前已确认的实体和关系包括：

| 类型 | 名称 | 含义 |
|---|---|---|
| Tag | `eval_suite` | 实验题库套件。 |
| Tag | `eval_group` | 题组。 |
| Tag | `eval_question` | 具体评估题目。 |
| Edge | `contains_group` | 套件包含题组。 |
| Edge | `contains_question` | 题组包含题目。 |

实验运行记录和人工评估记录由代理层 `experiment_service.py` 管理，主要用于对比实验页面展示、历史运行详情读取和评估状态保存。

## 论文可用表述

系统数据与知识层采用图数据库、知识库和文件化结果相结合的组织方式。设备故障知识以 Nebula 图空间存储，围绕设备、一级故障、二级故障、故障原因、现象、后果、风险、处置措施和应急资源建立实体与关系，为生成阶段提供可追溯的事实素材。预案模板同样存储在 Nebula 中，通过模板、版本和章节节点表达文档结构，并在章节节点中记录内容来源、图谱字段、固定文本和生成要求。案例资料则由 FastGPT 知识库承担向量检索，代理层根据当前设备和故障场景选择相应数据集。评估题库独立存储于 `llmkg_evaluation`，用于支撑对比实验和质量评价。通过上述分层组织，系统将故障知识、文档模板、案例经验和实验数据统一纳入生成链路，为运行时编排层提供稳定的数据基础。
