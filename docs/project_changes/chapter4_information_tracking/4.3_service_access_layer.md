# 4.3 服务接入层设计信息

本文档整理第4章“服务接入层设计”所需事实材料。服务接入层对应当前部署代理 `docker/frontend-proxy/server.py` 及其 `services/` 模块，主要负责前端鉴权、接口聚合、SSE 转发、子进程流水线调度和辅助能力接入。

## 信息来源

- `docker/frontend-proxy/server.py`
- `docker/frontend-proxy/services/pipeline_service.py`
- `docker/frontend-proxy/services/auth_service.py`
- `docker/frontend-proxy/services/case_search_service.py`
- `docker/frontend-proxy/services/template_service.py`
- `docker/frontend-proxy/services/quality_service.py`
- `docker/frontend-proxy/services/trace_service.py`
- `docker/frontend-proxy/services/experiment_service.py`

## 层级定位

服务接入层位于前端表现层和运行时编排层之间。它不直接承担大模型生成逻辑，而是负责将浏览器请求转换为后端流水线、图谱服务、FastGPT 插件和本地实验文件的调用。

当前部署代理使用 Python 标准库 `ThreadingHTTPServer` 启动，监听地址为 `0.0.0.0`，默认端口由 `FRONTEND_PROXY_PORT` 指定，默认值为 `8788`。

## 主要职责

| 职责 | 实现位置 | 说明 |
|---|---|---|
| 用户登录与会话 | `auth_service.py`、`server.py` | 基于用户名密码和 Cookie 维护登录态。 |
| 预案生成入口 | `server.py`、`pipeline_service.py` | 接收 `/api/pipeline/run`，调用 Python 流水线脚本。 |
| SSE 流式转发 | `pipeline_service.py`、`services/sse.py` | 将脚本 JSON 行事件转为浏览器 SSE 事件。 |
| 案例检索编排 | `case_search_service.py`、`pipeline_service.py` | 根据设备识别结果调用 FastGPT 知识库插件。 |
| 多故障参数转换 | `server.py`、`pipeline_service.py` | 将前端 `enableMultiFaultSearch` 转换为脚本 `--multi-fault`。 |
| 模板配置接口 | `template_service.py`、`server.py` | 查询和修改模板章节、提示词配置。 |
| 图谱溯源接口 | `trace_service.py`、`server.py` | 根据场景和图谱素材构造溯源子图。 |
| 质量评估接口 | `quality_service.py`、`server.py` | 支持格式优化、质量评估和结构化评价。 |
| 对比实验接口 | `experiment_service.py`、`server.py` | 支持实验运行、运行记录读取和评估记录保存。 |

## 预案生成请求处理

`/api/pipeline/run` 接收 JSON 请求体，核心字段包括：

| 字段 | 含义 |
|---|---|
| `question` | 用户输入的故障问题。 |
| `stream` | 是否使用 SSE 流式返回。 |
| `enableCaseSearch` | 是否并行执行案例检索。 |
| `enableMultiFaultSearch` | 是否启用多故障检索分支。 |

非流式模式下，代理层调用 `run_pipeline_sync()`，等待流水线结束后返回完整 JSON。流式模式下，代理层调用 `stream_pipeline()`，启动流水线子进程并逐行读取脚本标准输出。

子进程命令结构为：

```text
python scripts/run_parallel_generation_pipeline.py --question <问题> --output-dir <运行目录> --stream-events [--multi-fault]
```

每次请求会创建独立运行目录，目录名形如 `run_<毫秒时间戳>_<随机数>`。脚本结束后，代理层可从该目录读取 `pipeline_result.json`。

## SSE 转发机制

流式模式下，Python 流水线输出 JSON 行，每行包含 `event` 和 `data`。代理层读取后调用 `send_sse()` 写给浏览器。

关键事件包括：

| 事件 | 来源 | 前端用途 |
|---|---|---|
| `basic_info_started` | 流水线脚本 | 显示正在获取基本信息。 |
| `basic_info_done` | 流水线脚本 | 更新场景字段、图谱素材和边界判定。 |
| `template_split_started` | 流水线脚本 | 显示正在切分模板。 |
| `template_split_done` | 流水线脚本 | 初始化章节卡片。 |
| `parallel_generating_started` | 流水线脚本 | 显示正在并行生成章节。 |
| `chapter_started` | 流水线脚本 | 将章节状态置为生成中。 |
| `chapter_chunk` | 流水线脚本 | 将章节增量文本追加到对应章节。 |
| `chapter_done` | 流水线脚本 | 更新章节最终正文、耗时和状态。 |
| `pipeline_done` | 流水线脚本 | 保存完整生成结果。 |
| `pipeline_error` | 流水线脚本或代理层 | 显示错误或边界终止信息。 |
| `case_search_started` | 代理层案例线程 | 显示案例检索中。 |
| `case_search_done` | 代理层案例线程 | 展示案例检索卡片。 |
| `case_search_error` | 代理层案例线程 | 展示案例检索失败或跳过状态。 |
| `close` | 代理层 | 通知前端流结束。 |

代理层使用发送锁保护 SSE 写入，避免流水线事件和案例检索线程同时写响应导致数据交错。

## 案例检索编排

案例搜索不是 Python 主流水线内部逻辑，而是在服务接入层并发编排。若前端开启 `enableCaseSearch`，代理层会在基本信息阶段完成后，根据 `faultScene`、`graphMaterial` 和 `kbName` 推断对应数据集，再启动独立线程调用知识库案例检索插件。

案例检索插件输入为：

```json
{
  "当前识别设备": [{"datasetId": "..."}],
  "用户问题": "..."
}
```

代理层会从插件输出中提取最多 6 条案例卡片，并返回标题、知识库 ID、文档 ID、相关性和摘要。

## 鉴权设计

鉴权由 `auth_service.py` 实现。用户配置来自环境变量 `APP_LOGIN_USERS` 或 `APP_LOGIN_USERNAME`、`APP_LOGIN_PASSWORD`。登录成功后生成随机 session token，并通过 HttpOnly Cookie 写回浏览器。

| 配置项 | 默认值或含义 |
|---|---|
| `APP_LOGIN_COOKIE_NAME` | 默认 `llmkg_session`。 |
| `APP_LOGIN_SESSION_TTL` | 默认 `86400` 秒。 |
| 用户组 | 用户名为 `admin` 时为管理员，否则为普通用户。 |

会话存储在内存字典 `ACTIVE_SESSIONS` 中，并使用线程锁保护读写。

## 论文可用表述

服务接入层采用轻量级 Python HTTP 代理实现，作为前端页面与运行时编排、图谱服务、模板服务和实验服务之间的统一入口。该层通过登录 Cookie 维护用户会话，对除认证接口外的 `/api/` 请求进行鉴权，并将预案生成请求转换为 Python 流水线子进程调用。在流式生成模式下，代理层读取流水线输出的 JSON 行事件，将其转发为浏览器可消费的 SSE 事件，实现章节级增量展示。与此同时，案例检索、图谱溯源、模板配置、质量评估和对比实验均通过该代理层统一暴露给前端。该设计避免前端直接暴露 FastGPT 密钥、图数据库接口和本地文件路径，也使主生成流程与辅助能力能够在同一服务入口下协同运行。
