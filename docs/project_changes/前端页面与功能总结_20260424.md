# 前端页面与功能总结

本文基于当前仓库代码状态整理，聚焦前端页面、交互功能、主要数据流，以及每项功能的简要实现方式。

## 1. 总体结构

当前前端采用：

- `React + TypeScript + Vite`
- UI 组件库：`Ant Design`
- 图谱渲染：`@antv/g6`
- 统一入口文件：
  - [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx)

前端当前包含 4 个主页面路由：

- `/plan`：预案生成
- `/trace`：图谱溯源
- `/quality`：格式优化与质量评估
- `/template`：模板查看

另外还有：

- `/login`：登录页

## 2. 全局能力

### 2.1 登录鉴权

功能：

- 首次进入要求登录
- 顶部显示当前用户名和用户组
- 支持登出
- 未登录访问受保护接口时会被后端拦截

实现方式：

- 前端通过 [authApi.ts](/home/ubuntu/LLM-KG-database/frontend/src/services/authApi.ts) 调用：
  - `GET /api/auth/me`
  - `POST /api/auth/login`
  - `POST /api/auth/logout`
- 登录页组件：
  - [LoginPage.tsx](/home/ubuntu/LLM-KG-database/frontend/src/pages/LoginPage.tsx)
- 登录状态、当前路由和用户信息由 [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 统一管理

### 2.2 顶部页面导航

功能：

- 在 4 个主页面之间切换
- 保持统一顶部导航栏样式

实现方式：

- 路由状态由 [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 自己维护
- 使用 `history.pushState` / `popstate` 同步地址栏

### 2.3 显示标签开关

功能：

- 全局控制正文中的 `KG / GEN / FIX` 标签是否显示

实现方式：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 中维护 `showModeTags`
- 写入 `localStorage`
- 正文渲染时传给：
  - [RichTextRenderer.tsx](/home/ubuntu/LLM-KG-database/frontend/src/components/RichTextRenderer.tsx)
  - 以及页面内富文本渲染逻辑

### 2.4 紧凑布局开关

功能：

- 控制卡片布局密度
- 预案生成页可切换为一行 3 卡
- 质量评估页可切换为一行 4 卡

实现方式：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 中维护 `compactLayout`
- 写入 `localStorage`
- 页面组件按参数切换网格列数

### 2.5 夜间模式

功能：

- 整站暗色主题切换
- 页面头部、卡片、输入框、图谱容器、复选框等共同切换

实现方式：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 中维护 `darkMode`
- 通过 `document.body.classList.toggle('theme-dark', darkMode)` 控制
- 样式定义集中在：
  - [global.css](/home/ubuntu/LLM-KG-database/frontend/src/styles/global.css)

### 2.6 页面懒加载

功能：

- 降低首页首次加载压力

实现方式：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 对以下页面使用 `React.lazy`
  - `TraceGraphPage`
  - `QualityReviewPage`
  - `TemplateViewPage`

## 3. 预案生成页

页面位置：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx)

### 3.1 用户输入与运行流水线

功能：

- 输入故障问题/场景
- 点击“运行流水线”执行整套预案生成流程

实现方式：

- 前端调用：
  - [planApi.ts](/home/ubuntu/LLM-KG-database/frontend/src/services/planApi.ts)
  - `runPipelineStream(...)`
- 后端接口：
  - `POST /api/pipeline/run`
- 支持 SSE 流式事件：
  - 基本信息阶段
  - 模板切片阶段
  - 并行生成阶段
  - 案例检索阶段

### 3.2 章节级流式生成

功能：

- 每章独立显示
- 支持章节粒度状态更新
- 支持流式正文拼接

实现方式：

- [planApi.ts](/home/ubuntu/LLM-KG-database/frontend/src/services/planApi.ts) 负责解析 SSE 事件
- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 用 `onChapterStarted / onChapterChunk / onChapterDone` 更新章节数组

### 3.3 案例搜索开关

功能：

- 勾选后同时执行案例检索
- 返回案例卡片并展示在预案页面

实现方式：

- 请求参数中传 `enableCaseSearch`
- 后端在流水线基础上并发执行知识库案例搜索
- 返回 `case_search` 结构给前端

### 3.4 多故障检索开关

功能：

- 勾选后不走单故障识别链路
- 改走多故障识别与多故障图谱素材聚合链路

实现方式：

- 前端在请求体中传 `enableMultiFaultSearch`
- 后端代理将其转成脚本参数 `--multi-fault`
- 流水线脚本在基本信息阶段切换到多故障插件，并把：
  - `故障二级节点` 规范成数组
  - 自动补 `主故障二级节点`
  - 再逐故障调用图谱查询插件聚合 `graphMaterial`

相关实现：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx)
- [server.py](/home/ubuntu/LLM-KG-database/docker/frontend-proxy/server.py)
- [run_parallel_generation_pipeline.py](/home/ubuntu/LLM-KG-database/scripts/run_parallel_generation_pipeline.py)

### 3.5 填入示例

功能：

- 点击后随机填入预置问题
- 单故障模式使用普通问题库
- 多故障模式使用多故障问题库

实现方式：

- [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx) 内置：
  - `PRESET_QUESTIONS`
  - `MULTI_FAULT_PRESET_QUESTIONS`
- `handleFillExample` 根据 `enableMultiFaultSearch` 选择题库

### 3.6 结果持久化

功能：

- 一次生成完成后把当前问题与全部章节结果持久化
- 下次刷新后自动恢复

实现方式：

- 使用 `localStorage`
- key：
  - `llmkg_saved_plan_snapshot_v1`
- 读写逻辑在 [App.tsx](/home/ubuntu/LLM-KG-database/frontend/src/App.tsx)

### 3.7 正文渲染与清洗

功能：

- 把生成结果渲染为更像正式预案的富文本
- 支持标题、空行、标签、去元信息等清洗

实现方式：

- 页面内的 `renderMarkedText(...)`
- 全站组件：
  - [RichTextRenderer.tsx](/home/ubuntu/LLM-KG-database/frontend/src/components/RichTextRenderer.tsx)

### 3.8 导出与复制

功能：

- 复制全部章节结果
- 下载整个预案文本

实现方式：

- 复制：浏览器 clipboard API
- 下载： [download.ts](/home/ubuntu/LLM-KG-database/frontend/src/utils/download.ts)

## 4. 图谱溯源页

页面位置：

- [TraceGraphPage.tsx](/home/ubuntu/LLM-KG-database/frontend/src/pages/TraceGraphPage.tsx)

### 4.1 图谱查询入口

功能：

- 根据最近一次生成结果的 `question / faultScene / graphMaterial` 获取图谱子图

实现方式：

- 前端调用：
  - `POST /api/trace/subgraph`
- API 封装：
  - [planApi.ts](/home/ubuntu/LLM-KG-database/frontend/src/services/planApi.ts)

### 4.2 图谱布局与样式

功能：

- 根节点居中
- 一级、二级、三级、四级节点按分层圆环布局
- 节点为圆角矩形，无底色，彩色边框
- 文本在节点内显示

实现方式：

- 图谱场景和布局在：
  - [traceGraphScene.ts](/home/ubuntu/LLM-KG-database/frontend/src/pages/traceGraphScene.ts)
- 渲染库：
  - `@antv/g6`

### 4.3 图谱动画

功能：

- 根节点先出现
- 再按层级、按分支逐步展开
- 边先出现、节点后淡入

实现方式：

- 动画调度逻辑在：
  - [traceGraphAnimation.ts](/home/ubuntu/LLM-KG-database/frontend/src/pages/traceGraphAnimation.ts)
- 当前已改为增量更新节点/边状态，不再每帧重建整图

### 4.4 多故障图谱高亮

功能：

- 主故障保持焦点
- 所有命中故障共同高亮
- 命中一级故障分支按整分支展开，不只显示单个命中的二级故障

实现方式：

- 后端 `/api/trace/subgraph` 返回：
  - `isFocus`
  - `isHit`
- 后端会优先读取多故障 `graphMaterial / faultScene`
  - `主故障二级节点`
  - `故障二级节点[]`
- 前端动画优先渲染所有命中分支

### 4.5 左侧信息卡

功能：

- 显示设备根节点
- 显示主故障二级节点
- 显示命中二级故障列表
- 显示图谱节点数、边数、命中节点数

实现方式：

- [TraceGraphPage.tsx](/home/ubuntu/LLM-KG-database/frontend/src/pages/TraceGraphPage.tsx) 直接从 `trace.graph.nodes` 聚合

### 4.6 节点详情

功能：

- 点击图中节点，在侧栏显示名称、类型、描述

实现方式：

- 通过节点点击事件更新 `selectedNodeId`
- 再从当前 `trace.graph.nodes` 中查详情

## 5. 格式优化与质量评估页

页面位置：

- [QualityReviewPage.tsx](/home/ubuntu/LLM-KG-database/frontend/src/pages/QualityReviewPage.tsx)

### 5.1 原文 / 原文评估 / 优化评估三视图

功能：

- 左侧卡可切换：
  - 原文
  - 原文评估
  - 优化评估
- 右侧显示优化结果

实现方式：

- 每章维护多套状态：
  - 优化结果
  - 原文评估结果
  - 优化评估结果
- 使用 `Segmented` 和内部状态切换视图

### 5.2 单章与批量操作

功能：

- 单章：
  - 优化
  - 评估原文
  - 评估优化后
- 批量：
  - 全部优化
  - 全部评估原文
  - 全部评估优化后

实现方式：

- [QualityReviewPage.tsx](/home/ubuntu/LLM-KG-database/frontend/src/pages/QualityReviewPage.tsx) 内部自己做并发控制
- 批量并发数固定为 `6`

### 5.3 流式显示

功能：

- 先显示 reasoning
- reasoning 结束后切正式输出

实现方式：

- 请求：
  - `POST /api/quality/review`
- 前端按 SSE 事件分流：
  - `quality_status`
  - `quality_reasoning_chunk`
  - `quality_output_chunk`
  - `quality_done`

### 5.4 提示词管理

功能：

- 查看优化提示词、评估提示词
- 管理员可编辑保存
- 用户组只读
- 支持缓存与刷新

实现方式：

- 页面加载：
  - `/api/template/prompts`
- 保存：
  - `/api/template/prompt/save`
- 缓存到 `localStorage`

当前仍需要持续完善的点包括：

- 多故障图谱溯源页的进一步交互增强
  - 例如显式切换“主故障”
- 多故障生成提示词的持续优化

- 图谱动画性能仍可继续优化
  - 已改为增量更新，但大图仍有进一步压缩空间
- 前端状态与后端运行记录的统一持久化
  - 目前部分记录仍依赖本地缓存或临时产物
