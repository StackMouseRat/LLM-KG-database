## 当前工作流（基于 fastGPT_json/测试 Copy.json）

1. 工作流入口：接收用户问题。  
2. 缓存初始化分支：  
   - 若“一级/二级故障目录”变量为空，则先执行基础查询（一级-二级故障映射）。  
   - 通过代码节点清洗 HTTP 返回文本，生成可直接注入提示词的摘要文本。  
   - 将摘要写入全局变量，用于后续规划器稳定命中。  
3. 计划器节点：  
   - 基于用户问题 + 已缓存的一二级故障目录，输出分步查询计划。  
4. 循环节点：逐步执行每个计划步骤  
   - 循环提取：把当前步骤转成可执行查询体（JSON Body）。  
   - HTTP 请求：调用 Nebula HTTP 网关执行查询。  
   - 查询后处理：把查询结果转成自然语言结果段。  
   - 变量更新：累积每轮结果，形成有上下文的分步说明。  
5. 循环结束：输出聚合后的分步结果给后续回复节点。

3. 查询后处理可进一步统一模板，让每轮输出更符合一线运维口吻。
