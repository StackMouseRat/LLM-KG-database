# 输电线路导入问题与正确步骤（2026-04-15）

## 1. 本次背景
- 目标：将最新“输电线路”`xlsx` 图谱数据导入 `llmkg_transmission_line`。
- 最终结果：导入成功，`entity=190`、`rel=215`。

## 2. 这次耗时长的主要问题

### 问题 A：网关端口判断错误
- 现象：一直请求 `127.0.0.1:8787`，健康检查失败。
- 根因：当前容器映射端口为 `18787->8787`，不是本机 `8787`。
- 正确做法：
  1. 先查端口：`docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"`
  2. 再健康检查：`curl http://127.0.0.1:18787/graph/health`

### 问题 B：数据源自动定位不稳定（中文路径过滤）
- 现象：脚本初始阶段选不到正确输电线路目录，或匹配为空。
- 根因：Windows 终端/编码环境下，直接中文字符串匹配路径不稳定。
- 正确做法：
  1. 用 `Path.glob('**/*_nodes.xlsx')` 枚举。
  2. 用 `path.as_posix().encode('unicode_escape')` 做过滤匹配（如 `\u8f93\u7535\u7ebf\u8def`）。
  3. 最终锁定：`xls/成品/输电线路/输电线路_第一至第七层级图谱数据/`

### 问题 C：首批写入 502
- 现象：`INSERT VERTEX` 首批就 502。
- 根因：两类问题叠加：
  1. 单条 nGQL 过长（批次过大）。
  2. 文本中含非常规字符（私有区/控制字符）导致编码异常。
- 正确做法：
  1. 对文本字段做清洗：移除 `Cc/Cs/Co` 类字符（保留 `\n/\t`）。
  2. 缩小批次：
     - 顶点批次 `5`
     - 边批次 `10`
  3. 每批执行后立即检查 `ok`，失败时打印该批内容定位。

### 问题 D：调试打印触发 GBK 编码报错
- 现象：异常信息输出时触发 `UnicodeEncodeError`。
- 根因：控制台默认 GBK，直接打印原始异常体含特殊字符。
- 正确做法：
  1. 调试输出统一转义：`text.encode('unicode_escape').decode('ascii')`。
  2. 不直接原样打印含未知字符的长文本。

## 3. 正确步骤（推荐 SOP）

### Step 1：确认服务状态
1. 启动/拉起服务：
   - `docker compose -f nebula-docker-compose/docker-compose-lite.yaml up -d`
2. 确认网关端口（重点看 `nebula-http-gateway`）：
   - `docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"`
3. 健康检查：
   - `curl http://127.0.0.1:18787/graph/health`

### Step 2：锁定正确数据集
1. 仅在“输电线路”候选中选最新 `*_nodes.xlsx + *_links.xlsx` 配对。
2. 本次实际导入文件：
   - `xls/成品/输电线路/输电线路_第一至第七层级图谱数据/节点_nodes.xlsx`
   - `xls/成品/输电线路/输电线路_第一至第七层级图谱数据/关系_links.xlsx`

### Step 3：建表结构（幂等）
1. `CREATE TAG IF NOT EXISTS entity(name string, node_desc string, lvl int, source_id int, degree int, weight int, stroke string);`
2. `CREATE EDGE IF NOT EXISTS rel(relation string);`
3. `CREATE TAG INDEX IF NOT EXISTS entity_lvl_idx ON entity(lvl);`

### Step 4：数据预处理
1. 读取 `nodes/links`。
2. 字段映射：
   - 顶点：`id,name,desc,degree,weight,stroke`
   - 边：`from,to,relation`
3. 计算 `lvl`：
   - 用 `from->to` 构图，根节点 `in_degree=0`，BFS 得层级。
4. 文本清洗：
   - 删除 `Cc/Cs/Co` 字符，转义 `\` 和 `"`。

### Step 5：分批写入
1. `INSERT VERTEX entity(...) VALUES ...;`（批次建议 5）
2. `INSERT EDGE rel(relation) VALUES ...;`（批次建议 10）
3. 每批检查返回 `ok=true`，失败立即停止并定位该批。

### Step 6：结果校验
1. `MATCH (v:entity) RETURN count(v) AS c;` => 预期 `190`
2. `MATCH ()-[e:rel]->() RETURN count(e) AS c;` => 预期 `215`
3. 抽样查询层级：
   - `MATCH (v:entity) WHERE v.entity.lvl <= 2 RETURN v.entity.lvl, v.entity.source_id, v.entity.name ORDER BY v.entity.lvl, v.entity.source_id LIMIT 20;`
