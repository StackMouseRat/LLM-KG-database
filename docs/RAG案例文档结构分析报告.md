### 2. 案例边界识别问题

#### ❌ **问题1：案例边界不清晰**

**示例：电网设备故障典型案例_第1章_变压器**
```markdown
## 1.1 某 750kV 变电站 750kV 电抗器呼吸器溢油故障分析

### 一、故障情况
（案例描述）

## 二、初步分析处理
（分析过程）

三、停电检查处理情况
（处理过程）

四、检查处理结果分析
（结果分析）

五、整改措施及建议
（整改措施）
```

**问题**：
- "一、故障情况" 是 `### 三级标题`
- "三、停电检查处理情况" 是**普通文本**（无 # 标记）
- 同一个案例内部使用了中文序号（一、二、三、四、五）作为子章节
- 无法通过统一的标题层级识别案例边界

#### ❌ **问题2：案例内部结构不统一**

**示例：220kV设备异常_第四章_断路器**
```markdown
#### 一、220kV断路器弹簧机构不储能

1. 异常情况
（描述）

2. 分析处理
（处理）

二、110kV 线路断路器气动机构异常

1. 异常情况
（描述）

2. 分析处理
（处理）
```

**问题**：
- 案例标题是 `#### 四级标题` + 中文序号
- 案例内部使用阿拉伯数字（1. 2.）作为子章节
- 不同文档的案例标题层级不一致（有的是 ##，有的是 ####）

#### ❌ **问题3：中文序号混用**

在同一个文档中：
- 案例标题用：`一、二、三、...`
- 案例内部章节也用：`一、二、三、...`
- 无法区分哪个是案例边界，哪个是案例内部的子章节

## 二、当前结构是否满足"每案例切一片"？

### ❌ **结论：不满足**

### 原因分析：

1. **案例边界无法自动识别**
   - 没有统一的标题层级标记案例开始
   - 中文序号既用于案例标题，也用于案例内部章节
   - 无法通过正则表达式准确切分

2. **案例完整性难以保证**
   - 一个案例可能跨越多个标题层级
   - 案例内部的"一、二、三、四、五"是案例的组成部分，不应分割
   - 简单按标题切分会破坏案例的完整性

3. **文档结构不一致**
   - 17个文档使用了至少3种不同的结构模式
   - 无法用统一的规则处理所有文档

## 四、问题根源

### 1. 文档来源多样
- 来自不同的书籍、报告、手册
- 每个来源有自己的格式规范
- 没有统一的案例标记标准

### 2. Markdown 转换问题
- 原始文档可能是 PDF/Word
- 转换为 Markdown 时标题层级不一致
- 部分中文序号没有被识别为标题

### 3. 案例粒度不统一
- 有的案例很详细（3000+ 字）
- 有的案例很简短（400 字）
- 案例内部结构差异大

---

## 五、解决方案建议

### 方案1：手动标注案例边界（推荐）⭐

**步骤**：
1. 为每个案例添加统一的标记
2. 使用特殊标记符，如：`<!-- CASE_START -->` 和 `<!-- CASE_END -->`
3. 编写脚本按标记切分

**优点**：
- 准确率100%
- 保证案例完整性
- 可控制切片粒度

**缺点**：
- 需要人工标注（17个文件，估计200-300个案例）
- 工作量较大（约2-3天）

### 方案2：基于语义的智能切分

**步骤**：
1. 使用 LLM 识别案例边界
2. 提取案例标题、故障情况、处理过程、结论
3. 自动生成结构化 chunks

**优点**：
- 自动化程度高
- 可以理解案例语义
- 切分质量较好

**缺点**：
- 需要调用 LLM API（成本）
- 可能有识别错误
- 需要人工校验

### 方案3：混合标题层级 + 长度限制

**步骤**：
1. 识别 `##` 或 `###` 开头的标题作为潜在案例边界
2. 合并中文序号章节（一、二、三...）到同一个 chunk
3. 如果 chunk 超过1500字，按逻辑章节再切分

**优点**：
- 相对简单
- 可以快速实现
- 适合大部分文档

**缺点**：
- 准确率约70-80%
- 可能切分不完整
- 需要后期调整

### 方案4：按固定长度切分（不推荐）❌

**问题**：
- 会破坏案例完整性
- 一个案例可能被切成多片
- 检索时无法获得完整上下文

## 八、总结

### 当前状态
- ❌ 文档结构**不满足**"每案例切一片"的自动切分要求
- ⚠️ 案例边界不清晰，标题层级不统一
- ⚠️ 需要预处理才能用于 RAG

### 建议行动
1. **立即**：使用方案3快速切分，验证 RAG 效果
2. **短期**：根据效果决定是否需要手动标注或 LLM 切分
3. **长期**：建立案例文档标准化规范，统一未来文档格式

### 预期效果
- 方案3：准确率 70-80%，1周完成
- 方案1：准确率 95-100%，3周完成

**建议先用方案3快速验证 RAG 价值，再决定是否投入更多资源优化切分质量。**
