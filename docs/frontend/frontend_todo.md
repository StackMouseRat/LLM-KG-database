# 前端开发任务清单

## 1. 开发目标

在 3-5 天内完成论文演示版前端 MVP。

只实现：

- 预案生成
- 图谱溯源展示

## 2. 技术选型

- 前端框架：React
- UI 组件库：Ant Design
- 图谱可视化：AntV G6
- 请求方式：fetch
- 部署方式：Docker + Nginx
- 适配范围：PC 端

## 3. Day 1：项目骨架与接口联通

### 3.1 初始化项目

- 创建 React 项目
- 安装依赖：
  - `antd`
  - `@antv/g6`
  - `dayjs`
- 配置基础样式
- 配置环境变量：
  - `VITE_API_BASE_URL`

### 3.2 页面框架

- 创建主布局
- 顶部标题：
  - `电力设备应急预案生成系统`
- 创建 Tab：
  - `预案生成`
  - `溯源分析`

### 3.3 API 联通

- 创建 `services/planApi.ts`
- 实现 `generatePlan(question)` 方法
- 先使用非流式接口跑通：

```http
POST /api/plan/generate
```

### 3.4 验收标准

- 页面能打开
- 输入问题后能调用接口
- 能展示返回的预案文本

## 4. Day 2：预案生成页

### 4.1 输入区

- 多行文本框
- 默认示例问题
- 清空按钮
- 生成按钮

### 4.2 输出区

- Markdown 风格展示
- 支持滚动
- 支持加载态
- 支持错误态

### 4.3 状态提示

不做百分比进度，只做阶段状态：

- `正在识别设备`
- `正在查询知识图谱`
- `正在生成预案`
- `生成完成`

### 4.4 操作按钮

- 复制
- 下载 Markdown
- 查看溯源

### 4.5 验收标准

- 输入故障描述后生成方案
- 输出不重复
- 复制功能可用
- 下载 Markdown 可用

## 5. Day 3：溯源分析页

### 5.1 图谱数据处理

从接口返回的 `trace.graph` 中读取：

- `nodes`
- `edges`

如果接口没有直接给 `trace.graph`，则从 FastGPT `detail.responseData` 中解析：

- `设备识别`
- `故障类型分析`
- `下游节点获取`
- `下游节点数据清洗`

### 5.2 G6 图谱展示

- 初始化 G6 图
- 使用层级布局或力导向布局
- 按节点类型设置颜色：
  - 二级故障
  - 故障原因
  - 故障现象
  - 应对措施
  - 故障后果
  - 安全风险
  - 应急资源

### 5.3 节点详情面板

点击节点后展示：

- 节点名称
- 节点类型
- 节点描述
- 来源标记

### 5.4 验收标准

- 能显示本次生成涉及的图谱节点
- 节点可点击
- 详情面板能显示描述

## 6. Day 4：联调与稳定性

### 6.1 多设备测试

至少测试：

- 高压断路器
- 电力电缆
- 变压器
- 输电线路
- 互感器

### 6.2 异常处理

- FastGPT API 失败
- Nebula 网关失败
- 设备识别为空
- 图谱结果为空
- 输出为空

### 6.3 溯源跳转

- 在预案生成页点击 `查看溯源`
- 自动切换到 `溯源分析`
- 展示当前请求的图谱数据

### 6.4 验收标准

- 至少 3 类设备可稳定演示
- 出错时界面不崩溃
- 可从生成结果跳转到溯源页

## 7. Day 5：答辩版收尾

### 7.1 UI 优化

- 统一颜色
- 优化空状态
- 优化 loading
- 优化按钮位置

### 7.2 示例问题

内置 3 个示例：

- 电缆沟进水导致开关柜绝缘告警
- 高压断路器合分闸线圈故障
- 主变油温异常升高

### 7.3 Docker 化

- 编写 `Dockerfile`
- 编写 `nginx.conf`
- 可选：编写 `docker-compose.yml`

### 7.4 演示材料

- 截图：
  - 输入页
  - 生成结果
  - 溯源图谱
  - 节点详情
- 准备 1 条稳定演示样例

## 8. 最低验收标准

- 能输入场景并生成应急预案
- 能复制和下载方案
- 能展示图谱溯源
- 能点击节点查看详情
- 能稳定演示至少 3 类设备
